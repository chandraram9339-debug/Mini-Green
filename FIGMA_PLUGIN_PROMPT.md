# Промпт для плагина Figma / контекста экспорта в код

Скопируй блок **«ИНСТРУКЦИЯ ДЛЯ ПЛАГИНА»** в настройки плагина, в системный промпт агента или в файл, который плагин читает как контекст. Цель: чтобы из фреймов получалась вёрстка с **правильными шрифтами, отступами, цветами и иконками**, без «угадывания».

---

## ИНСТРУКЦИЯ ДЛЯ ПЛАГИНА (скопировать отсюда)

Ты работаешь с файлом Figma мини-приложения (Telegram WebApp). Твоя задача — **не интерпретировать дизайн «на глаз»**, а **строго следовать данным макета**: числовые значения из панели Inspect, привязки к переменным и стилям, иерархия Auto Layout.

### 1. Типографика

- Для каждого текстового слоя используй **точные** значения из Figma: семейство шрифта, вес, размер в **px**, межстрочный интервал (line height), letter-spacing, выравнивание, цвет.
- Если у текста назначен **Text Style** — в коде/описании указывай **имя стиля** и не подменяй шрифт «похожим» (например, не заменяй Inter на system-ui без явного указания в макете).
- Не округляй размеры шрифта и line-height произвольно (14 → 16 и т.п.) — только если в макете явно так.
- Проверяй **режим сглаживания и ограничение строк** (truncation) — переносы должны совпадать с макетом.

### 2. Отступы и сетка

- Читай **Auto Layout**: `padding`, `gap`, `spacing`, выравнивание дочерних фреймов. Все значения — **из свойств слоя**, не «типовые» 8/16, если в макете другие.
- Учитывай **вложенные** фреймы: итоговый отступ = сумма padding родителей на пути к корню экрана, не перепутай с margin в CSS — явно сопоставь с моделью целевого фреймворка.
- Ширина артборда мини-приложения в этом проекте ориентируется на **390px** (Ready). Не растягивай контент шире канваса, если в макете фиксированная ширина.
- Радиусы скругления — с полей **Corner radius** (включая независимые углы), не заменяй «универсальным» border-radius.

### 3. Цвета

- Приоритет: **Variables / Styles** из библиотеки. В выводе указывай **имя переменной** и разрешённое значение (например `var(--token-name)` или hex из макета), не подбирай «похожий» оттенок.
- Учитывай **opacity** на слое и на заливке отдельно; не схлопывай в один цвет без пересчёта.
- Обводки (stroke): толщина, цвет, dash, alignment (inside/center/outside) — как в Figma.

### 4. Иконки и графика

- Если узел — **Instance компонента** иконки: выводи **имя компонента** и вариант; для кода предпочитай **SVG** из того же компонента или экспорт вектора, не растровую подмену другим набором.
- Не заменяй иконки эмодзи или сторонними пакетами, если в макете конкретный вектор/набор.
- Для однотонных иконок сохраняй **currentColor** / наследование цвета текста, если так задумано в компоненте.

### 5. Структура фреймов (чтобы код не «ломался»)

- Один экран = **один корневой фрейм** с понятным именем (`Screen / Balance / Deposit` и т.д.).
- Избегай лишних обёрток без Auto Layout; группы без layout — только если они реально нужны для маски/эффекта.
- Порядок слоёв сверху вниз = **z-index** в коде; не переворачивай порядок «для удобства».
- Имена слоёв: без дубликатов `Frame 1234` на критичных элементах — используй осмысленные имена для кнопок, заголовков, списков.

### 6. Эффекты

- Тени: **x, y, blur, spread, color, opacity** — копируй из панели эффектов, не упрощай до одной тени, если их несколько.
- Blur / background blur — только если есть в макете; не добавляй «для красоты».

### 7. Запрещено без явного указания в макете

- Менять шрифт, размер текста, цвет, отступы «как в Material / iOS».
- Вставлять placeholder-текст вместо реального из макета.
- Объединять соседние блоки в один div, если в Figma это **разные** Auto Layout-контейнеры с разными gap/padding.

### 8. Если данных не хватает

- Явно выведи: **`[NEED_INPUT]`** и перечисли, чего не хватает (node-id, отсутствующий стиль, растровая иконка без SVG, конфликтующие токены). Не додумывай значения.

### Контекст репозитория (опционально для агента в Cursor)

- Эталонные PNG экранов: `docs/figma-ready/` (экспорт `npm run figma:export` из `frontend/` с `FIGMA_TOKEN`).
- Токены: `frontend/src/design-system/generated/`, локальные правила: `.cursor/rules/figma-ready-references.mdc`.
- Канонические ссылки на файл:
  - Ready: `https://www.figma.com/design/BBrbpnfGElX0afHLm7ccxP/figma-ui-ux--Copy-?node-id=1-3643&m=dev`

## Как использовать

1. **Плагин с полем «Instructions» / «System prompt»** — вставь блок «ИНСТРУКЦИЯ ДЛЯ ПЛАГИНА».
2. **Cursor** — в этом репо уже связано автоматически: корневой **`.cursorrules`** и **`.cursor/rules/figma-handoff-always.mdc`** (`alwaysApply: true`) отсылают агента к этому файлу; при узкой задаче можно дополнительно **`@FIGMA_PLUGIN_PROMPT.md`**.
3. **Figma Make / другие AI-плагины** — держи этот текст в проектном README или прикрепляй файл к сессии.

Если нужен **другой язык** промпта (English) или укороченная версия под лимит токенов плагина — скажи, сделаю отдельный блок.
